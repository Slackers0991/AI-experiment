## 1. Назначение
Автоматические проверки, которые предотвращают деградацию качества и архитектуры.

## 2. Обязательные проверки
- format/lint
- unit tests
- contract tests
- dependency rules (no cycles, layer violations)
- минимальный coverage (если применяется)
- **documentation validation** (проверка структуры и трассировки документов)
- static analysis (optional)

## 3. Правила архитектуры, которые должны проверяться
- Domain не импортирует инфраструктуру
- Presentation не импортирует DTO
- UI не импортирует Data
- ...

## 4. Проверка документации

### 4.1 Валидация структуры документов
**Инструмент:** `scripts/validate_docs.py`

**Проверяет:**
- Наличие обязательных разделов в шаблонах (Artifact ID, Traceability)
- Корректность формата идентификаторов (FEATURE-001, SCREEN-001, SLICE-001)
- Заполнение обязательных полей трассировки
- Корректность ссылок на идентификаторы в трассировке
- Соответствие Screen ID в трассировке с Artifact ID

**Команда для локальной проверки:**
```bash
python scripts/validate_docs.py
```

**Интеграция в CI:**
```yaml
# Пример для GitHub Actions
- name: Validate documentation
  run: python scripts/validate_docs.py
```

**Блокирует merge при:**
- Отсутствии обязательных разделов
- Некорректном формате идентификаторов
- Незаполненных обязательных полях трассировки
- Некорректных ссылках на несуществующие артефакты

### 4.2 Правила валидации
См. [`docs/validation_rules.md`](../validation_rules.md) для детального описания правил валидации.
