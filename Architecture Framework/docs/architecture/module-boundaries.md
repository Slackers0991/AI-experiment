## 1. Назначение
Стандарт структуры модулей и правила зависимостей. Это регламент, который обеспечивает управляемость.

## 2. Структура проекта (рекомендуемая)
```
app/
core/
features/
  <feature>/
```

## 3. Структура feature (обязательный шаблон)
```
features/<feature>/
  ui/
  presentation/
  domain/
  data/
```

## 4. Правила зависимостей (в одну сторону)
- ui → presentation
- presentation → domain
- data → domain
- domain → (никто, кроме общих типов)

Запрещено:
- ui → data
- presentation → data
- domain → data/infrastructure
- feature → feature напрямую (только через API/ports или navigation)

## 5. Взаимодействие между features (разрешённые способы)
- Navigation (routes/deeplinks)
- Public API/Port (features/<x>/api) — если применяется
- Shared types (core/domain) — только универсальные

## 6. Как проверяем соблюдение (quality gates)
- Список инструментов/линтеров/проверок
- CI шаги
