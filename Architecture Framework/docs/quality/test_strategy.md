## 1. Цель
Определить, что чем покрываем и по каким критериям принимаем фичу/экран.

## 2. Пирамида тестов (целевые доли)
- Unit: (use cases, reducers, mappers)
- Contract: (DTO↔Domain, error mapping, golden samples)
- Integration: (repositories, cache/network integration)
- E2E: (критические пользовательские пути)

## 3. Минимальные требования (DoD)
- Все новые use cases покрыты unit-тестами
- Все reducers/store покрыты тестами переходов + эффектов
- Все новые DTO имеют golden sample + contract test
- Линтеры/форматтеры проходят
- Обновлены документы (feature spec / state machine / contracts) при изменении поведения

## 4. Матрица покрытия (пример)
| Artefact / Rule | Test type | Location | Notes |
|---|---|---|---|
| Cache policy | integration + unit | ... | ... |
| Degradation rules | unit | ... | ... |
