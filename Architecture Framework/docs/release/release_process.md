# Release Process & Versioning

<!-- ИНСТРУКЦИЯ ПО ЗАПОЛНЕНИЮ
Цель: Зафиксировать процесс релиза, версионирование и стратегию rollout. Это регламент для выпуска версий приложения.

Когда заполнять: На Этапе 6.3 один раз для проекта. Обновляется при изменении процесса релиза или требований к версионированию.

Исходные данные:
- Процесс разработки команды (ветки, workflow)
- Требования к версионированию
- Стратегия feature flags
- Требования к changelog

Где брать:
- Git workflow команды
- Требования к версионированию (Semantic Versioning, и т.д.)
- Стратегия rollout (постепенный релиз, A/B тесты)
- Требования продукта к changelog

Советы:
- Версионирование должно быть явным — какой формат используем (SemVer, и т.д.)
- Правила обратной совместимости должны быть понятными — что можно менять, что нельзя
- Процесс релиза должен быть пошаговым — чеклист помогает не забыть шаги
- Feature flags должны быть частью стратегии — как включаем/выключаем фичи
- Rollout strategy должна быть безопасной — постепенный релиз, возможность отката
- Changelog должен быть полезным для пользователей — понятные описания изменений
- Процесс должен быть понятен всем членам команды — документируйте edge cases
-->
